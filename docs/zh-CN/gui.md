# 向导 GUI 指南

[English](../gui.md) | [简体中文](gui.md)

`v0.1.1` 向导是由配置驱动的 Tkinter 界面，使用与 CLI 相同的生成管线。
Windows 是主要经过测试的 GUI 平台。

## 启动 GUI

在 Windows 上，先运行 `install_requirements.bat` 安装依赖，再使用
`run_gui.bat` 启动应用。

等价的 PowerShell 命令：

```powershell
python -m pip install -r requirements.txt
$env:PYTHONPATH = "src"
python -m nbs2func.gui.app
```

步骤栏会显示已完成、当前和锁定的步骤。可以点击已完成步骤而不丢失配置值。
每次导航都会先应用并验证当前步骤。

## 语言

GUI 支持英文和简体中文。首次启动时，如系统语言受支持则使用系统语言，
否则回退到英文。从“语言”菜单选择的语言会保存到
`~/.nbs2func/gui_settings.json`，供以后启动时恢复。

切换语言会重建可见的向导文本，但不会替换 `WizardState`、重新读取 NBS
文件或重新开始生成。当前页面、此前已解锁步骤、Generate 解锁状态、歌曲
摘要、配置和已完成的生成输出都会保留。当前页面上的有效修改会在重建前
应用；若修改无效，语言切换会被取消，菜单恢复为原语言，页面和草稿保持
不变。生成运行期间，“语言”菜单不可用。

CLI 命令、配置字段名和内部 enum 值仍保持英文。

## 1. 输入

使用“浏览”选择 `.nbs` 文件，或输入路径后选择“加载 NBS”。读取成功后会
显示歌曲名称、长度、速度、图层数、音符数和可用乐器摘要。

如果路径改变，向导会在继续前重新加载并验证。空路径、文件不存在、目录
路径或格式错误的 NBS 都会阻止导航，不会复用旧歌曲摘要。本地 Unicode
文件名（包括中文和日文）受支持。

## 2. 布局

选择一种布局策略：

- `basic_linear`（基础线性布局）：把一个选定轨道排成直线。
- `track_based_stereo`（基于音轨的立体声布局）：根据音量和声像为图层/轨道
  确定固定位置。
- `note_based_stereo`（基于音符的立体声布局）：逐音符进行轨道槽位放置；
  默认的启发式预览模式。

## 3. 布局选项

设置轨道方向、世界原点 X/Y/Z、精确 Minecraft 版本，以及与所选布局相关
的选项。

方向可选东（+X）、西（-X）、南（+Z）和北（-Z）。GUI 会按选定版本配置
验证原点 Y，并在上下两侧各保留 48 格声音传播距离。GUI 不提供手动
`min_world_y` 或 `max_world_y` 覆盖。

基于音轨的立体声布局会显示面向用户的距离控制，固定的碰撞与中央拆分行为
保持自动。基于音符的立体声布局提供安全、均衡、密集和自定义配置；预设值
只读，选择自定义后才可编辑高级候选参数。

## 4. 模块

模块页包含启动模块、播放辅助和速度控制。

- 启动模块创建同步启动命令方块的位置。
- 播放辅助添加基于矿车的运行时逻辑、玩家记分板状态和可选的准备/开始按钮。
- GUI 中的播放辅助依赖启动模块。
- 速度命令模式依赖播放辅助。

启动原点和命令模块原点会根据布局方向验证。播放音乐起点跟随布局原点。
已禁用模块的字段不会参与验证。

速度模式为 `none`、`report` 和 `command`；后端为 `auto`、`carpet` 和
`vanilla`。仅报告模式不会改变世界 tick rate。生成命令模式会写入开始/
重置命令，并要求相应的服务器权限与后端能力。

## 5. 输出

选择一种输出格式：

- `datapack`（数据包（datapack）/ mcfunction）：完整方块构建及已启用的
  运行时模块。
- `schem`（原理图（.schem））：仅结构的 `.schem`；不能使用依赖运行时的
  模块。
- `both`（两者都生成）：完整方块原理图加仅运行时的 datapack 命令。

Datapack 字段包括输出路径、支持 Unicode 的文件夹名、命名空间和构建方式。
原理图字段包括输出路径、文件名和原点模式。默认原理图名称来自输入 NBS 的
主文件名，并保留 Unicode。

Datapack 构建方式：

- 简单函数链把大型输出拆分为直接相连的函数文件，不进行传送、区块等待、
  玩家传送窗口或计划延迟。
- 玩家传送分段构建使用空间窗口，把已配置的构建玩家传送过去，等待区块，
  并按计划运行命令部分。它是默认方式，推荐用于大型结构。

若启用了启动模块或播放辅助，则不能只输出原理图，因为这些模块需要运行时
逻辑。要同时获得原理图方块和运行时命令，请选择 `both`。

## 6. 摘要

检查输入歌曲、布局、重要选项、输出格式与路径、模块、速度模式和警告。
“保存配置”会把当前 `Nbs2FuncConfig` 写入 JSON。底部的“生成”按钮会
应用摘要状态、执行最终验证、确认任何已有 datapack 文件夹，然后进入生成
页面。保存配置只写出配置，不会开始生成；“生成”才会进入生成流程。

## 7. 生成

生成在后台 Python 线程中运行，并向 Tk 主循环报告结构化事件。页面显示：

- 单调递增的总进度；
- 会被覆盖更新而不是重复追加的当前阶段进度；
- 简洁的阶段、提示、警告、输出、完成和错误日志；
- 打开 datapack/mcfunction 文件夹和打开原理图文件夹操作；
- “生成另一个”、“返回”和“完成”操作。

生成运行时，导航、“完成”和会改变状态的菜单均不可用。生成完成或失败后，
“返回”会回到摘要；“完成”使用正常关闭处理器。“生成另一个”会清除生成
状态，并使用默认配置返回输入页。

生成期间关闭窗口或选择“文件 > 退出”时，GUI 会要求明确确认，因为 worker
无法安全取消，且可能留下不完整文件。

对于 `datapack` 和 `both`，若目标 datapack 根目录已存在，GUI 会在生成前
询问。拒绝后向导仍可编辑，不会启动 worker 或删除文件。仅输出原理图时不会
显示此 datapack 提示。

本预览版仅在 Windows 上支持打开输出文件夹。

## 配置菜单

- “新建”把向导和 GUI 专用状态重置为默认值。
- “加载配置”从 JSON 创建新的向导状态，并尝试重新加载输入歌曲。
- “保存配置”和“配置另存为”都会在写入前应用当前页面。
- “退出”使用与窗口关闭按钮相同的关闭处理器。

GUI 没有独立的生成实现，也不会解析 CLI 标准输出。
