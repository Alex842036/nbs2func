# 已知问题与预览限制

[English](../known_issues.md) | [简体中文](known_issues.md)

`v0.1.1` 可以使用，但仍处于实验阶段。运行生成的构建函数前，请备份目标
Minecraft 世界。

- `note_based_stereo` 仍采用启发式算法。困难歌曲可能无法产生理想排列。
- 非常大的歌曲可能需要大量处理时间和内存。当前布局生成器大体为单线程，
  因此在多核系统上总 CPU 利用率可能看起来较低。
- CPU 密集型生成会与 Tkinter 主线程共享 CPython 的 GIL，因此即使总 CPU
  利用率较低，GUI 也可能短暂无响应。
- 生成期间没有安全的取消操作。
- 生成期间关闭程序可能终止 daemon worker，并留下不完整输出文件。GUI 会
  在关闭前发出警告。
- 简单函数链输出不会加载区块，也不会在函数文件之间等待。请保持整个目标
  区域加载。
- 玩家传送输出需要有效且在线的构建玩家。断开连接、切换维度或中断命令可能
  留下不完整构建。
- GUI 主要在 Windows 上测试；macOS/Linux GUI 行为尚未完整验证。
- 本预览版的打开输出文件夹功能仅支持 Windows。
- 仅原理图输出无法包含依赖运行时的启动模块或播放辅助行为。
- 原理图文件不会把召唤的盔甲架或矿车作为存活原理图实体嵌入；在 datapack/
  `both` 工作流中由运行时命令创建。
- 重新生成时会替换已有的 nbs2func 构建函数文件。GUI 在复用已有 datapack
  文件夹前会询问；CLI 自动覆盖。
- 不支持的乐器或基底方块会令生成失败，不会静默回退到其他声音。
- 不支持 Minecraft 基岩版。
- 本版本尚未实现手动轨道/分组排列。
- 没有交互式 2D/3D 布局编辑器或预览。
- 尚未实现服务器辅助和 RCON 工作流。

速度计算已经实现。默认 `report` 模式只建议 tick rate，不会改变世界。
`command` 模式需要播放辅助、适当权限，以及选定的兼容 Carpet 或原版后端。
较旧的受支持配置不原生提供 1.21.1 的原版命令配置。

运行 `/function nbs:build/start` 前，请确认所选精确 Minecraft 配置、命名
空间/构建路径、已配置的玩家名、输出位置以及所需服务器权限或模组。
